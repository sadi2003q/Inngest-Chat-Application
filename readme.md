## üåü About The Project

Welcome to the Inngest Chat Application, a powerful demonstration of how to build a modern, event-driven, and intelligent chat interface. This project is more than just a chat app; it's a blueprint for creating scalable applications that leverage Large Language Models (LLMs) like Google's Gemini.

At its core, this application provides a seamless, real-time conversation with an AI. But behind the scenes, it showcases a robust architecture using **Inngest** to orchestrate complex workflows, ensuring every message is processed reliably. The frontend is a sleek, responsive experience crafted with **React** and **Vite**, styled with the utility-first power of **Tailwind CSS**. User identity and data persistence are managed securely through **Firebase**.

Whether you're a developer looking to understand AI integration, event-driven architectures, or modern frontend practices, this project serves as a comprehensive, hands-on guide.

### Architectural Flow

The application's architecture is designed for clarity and scalability. Here‚Äôs a high-level overview of how data flows:

1.  **User Interaction (View)**: A user sends a message through the React-based UI.
2.  **Business Logic (Controller)**: The corresponding controller handles the input, updates the local UI state, and prepares to call the backend.
3.  **Event Trigger (Inngest)**: Instead of a direct API call, the controller sends an event to Inngest (e.g., `"gemini/chat.request"`). This decouples the frontend from the backend logic.
4.  **Backend Processing (Express + Inngest)**: An Express server hosts the Inngest functions. The appropriate function listens for the event, picks up the job, and securely calls the Google Gemini API.
5.  **AI Response**: The Gemini API processes the request and returns the AI-generated message.
6.  **Data Persistence (Firebase)**: The conversation is saved to Firestore for long-term storage and context.
7.  **Real-time Update**: The response is pushed back to the client, where the UI updates in real-time.

This event-driven approach means the frontend doesn't need to wait for the AI to respond. It can move on, and the UI will be updated whenever the response is ready, leading to a non-blocking, highly responsive user experience.

## üöÄ Getting Started

Ready to dive in? Follow these steps to get the project running on your local machine.

### ‚úÖ Prerequisites

Make sure you have the following software installed:

-   **Node.js**: v18 or later. You can check your version with `node -v`.
-   **npm**: Should be included with Node.js.
-   **Google Gemini API Key**: Get one from the [Google AI Studio](https://aistudio.google.com/app/apikey).
-   **Firebase Project**: You'll need a Firebase project to handle authentication and database storage.

### üõ†Ô∏è Installation & Setup

1.  **Clone the Repository**
    ```bash
    git clone https://github.com/sadi2003q/Inngest-Chat-Application.git
    cd inngest-chat-application
    ```

2.  **Install Dependencies**
    This will install all the necessary packages for both the frontend and backend.
    ```bash
    npm install
    ```

3.  **Set Up Firebase**
    If you don't have a Firebase project, create one for free in the [Firebase Console](https://console.firebase.google.com/).
    -   Create a new project.
    -   In the project dashboard, add a new **Web App**.
    -   Go to **Project Settings** > **General**, and find your web app's configuration object. You'll need these values for your environment variables.
    -   In the console, go to **Authentication** > **Sign-in method** and enable the **Email/Password** provider.
    -   Go to **Firestore Database** and create a new database. Start in **test mode** for easy setup (you can secure it later with security rules).

4.  **Configure Environment Variables**
    Create a file named `.env` in the root of the project and paste the following content. Replace the placeholder values with your actual keys and credentials.

    ```env
    # .env

    # For Vite (Client-side) - Must be prefixed with VITE_
    VITE_GEMINI_API_KEY=your_gemini_api_key_here
    VITE_FIREBASE_API_KEY=your_firebase_api_key
    VITE_FIREBASE_AUTH_DOMAIN=your_firebase_auth_domain
    VITE_FIREBASE_PROJECT_ID=your_firebase_project_id
    VITE_FIREBASE_STORAGE_BUCKET=your_firebase_storage_bucket
    VITE_FIREBASE_MESSAGING_SENDER_ID=your_firebase_messaging_sender_id
    VITE_FIREBASE_APP_ID=your_firebase_app_id
    VITE_FIREBASE_MEASUREMENT_ID=your_firebase_measurement_id

    # For Express Server (Server-side)
    GEMINI_API_KEY=your_gemini_api_key_here

    # Optional: For Inngest Cloud signed events
    # INNGEST_EVENT_KEY=your_inngest_event_key
    ```
